计算机系统: 学习笔记

Posted on 2020-05-15 | In 计算机 |

听说卡耐基梅隆大学的计算机系统课程 CS 15213 课程十分出名, 遂在知乎上了解了一番. 看到一篇”在CMU上镇校神课15213是一种什么样的体验？” 下的回答, 了解到了这个课程的主要内容, 以及其十分特色的大作业系统 “Lab”. 每一个 Lab 就是一个大作业, 8个 Lab 覆盖了计算机系统的 8 个十分重要的主题. 其中的作业方式也是十分有趣, 其中我最喜欢的大概是 Lab 3 (or 4), 反向拆炸弹的大作业. 据这个回答所说, 整个做作业的过程就像是在玩游戏一样, 惊险刺激.

这次的任务很『简单』，一共有七关，六个常规关卡和一个隐藏关卡，每次我们需要输入正确的『拆弹密码』才能进入下一关，而具体的『拆弹密码』藏在汇编代码中。进入隐藏关卡的方式也在其中！这就需要我们一点一点探索蛛丝马迹了.

据另外一位答主的评论

前面几次非常底层的作业，每个人的答案都是自动生成的且不同的，除了弄清楚背后的原理，没有任何捷径可以走
所有作业的设置，如果弄清楚了每个重要知识点，加上一点点自己的联想和思考，思路就会非常清晰的，不会出现因为文档或者助教质量而导致的各种问题，也就是说，你在这门课上花的时间，大部分是有价值的
知识点不局限于课程大纲，深入浅出，承前启后，无论之后想往哪个方向发展，都可以从中受益
期中和期末考试是对日常作业的补充，重要的是理解而非记忆

这种每分每秒都能感受到老师用心的课程，又怎么能不是『神课』呢？

看上去, 大家都把这门课夸到天上去了啊. 是不是真的是这样? 我便是抱着这样的心态入坑的. 在 B 站上搜了搜 CMU 15213, 找到了 2015 Fall 与 2015 Spring 的课程, 随便选了一个便开始了学习计算机系统的旅程.

这篇文章的内容主要是记录我在学习 CMU 15213 过程中的学习笔记, 学习心得, 疑问, 以及各个 Lab 作业的记录. 将不定期更新.
主要参考文献是 Randal Bryant, David O’Hallaron. Computer System: a Programmer’s Prospective 以及 Brian Kernighan. The C Programming Language.

关于整数表示

这里记录的是 Randal Bryant, David O’Hallaron. Computer System: a Programmer’s Prospective (CSAPP) Chap 2.2 - 2.3 的学习心得.

由于 C/C++ 中包含了 Unsigned 与 Signed 的证书类型, 因此在使用他们的时候需要格外小心. 这里有几个问题是需要注意的

  1. Unsigned 和 Signed 相加与相乘问题, 逻辑运算问题: 当两个不同类型的整数相互操作的时候, 会有什么样的结果? 例如 “1u+1” 最终出来的数是Unsigned 还是 Signed? 还有 1u > -1 的结果是 True 还是 false?
  2. Unsigned 或 Signed 相加的溢出问题: 由于 Unsigned int 和 Signed int 都是 4 bytes 的数据, 当相加时有可能会溢出, 那么溢出之后的结果是多少? 如何确定?

而对于第二个问题, CSAPP 上详细讲了一大堆内容, 并且每个 Claim 都给出了十分冗长的证明. 在我看来十分没有必要, 实际上一个图就可以搞定所有的 Claim.

两类整数相加有几个特点, 以 4 bits 为例:

  • 循环性: 对于 Unsigned 来说, 按顺序 0, 1, 2, 3, …, 15, 0, 1. 即每个数都比前一个数+1, 最后会回到原点. 即 Unsigned 这个数据结构形成了一个圆环的结构: 绕了一圈回到了原点. 可以用一个圆环来描述这种类型的整数. 同样的, 对于 Signed 来说, 按顺序: 0, 1, …, 7, -8, -7, … 0, 1. 同样是一个圆环的结构.

  • 十进制的突变性: 这个仅体现在 Signed 的整数中, $7+1=-8$, 只在这个位置发生了十进制数值的突变.

基于第一个特点, 实际上我们完全可以用一个圆环来想象这两种整数类型.
